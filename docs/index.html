What’s New

The “2019 update that isn’t really an update” edition.

While attempts have been made to make a UserCSS version of ss16, due to full-time employment, this userstyle has’t seen the updates I would like.

The 20XX branch does have some changes, mainly to adapt to the new 4chan/4channel domain split. But after the whole ISPs-banning-4chan and needing a VPN in my country, I’ve slowly drifted away from it all.

If I find the time, I will try to get a v2.1.0 released.

v2.0.0 a.k.a. ss18 has now been released.

Fortunately (or unfortunately, depending on your view), ss16, remains much the same after 2 years. A conflict with CCleaner and Stylus meant that ss18 had to be completely rewritten. But, this lead itself to some small improvements.

Since Stylus is now the preferred userstyle manager, we can make use of UserCSS, so ss16 can now be auto-updated. One caveat is it does not allow the use of @import, so for font loading to work, users must also install the complimentary font userstyle.

Some changes include:

  • Better Custom Property naming, meaning variables make more sense.
  • The catalogue now makes use of display:grid as opposed to Masonry. Good in one way, bad in another.
  • sidedish uses ES6 to try and be different.
  • Board banners are supported via, again, an additional userstyle.

Future versions should hopefully include:

  • OneeChan support, meaning your favourite anime “girl” can follow you as you browse.
  • Complete 4chan X support (all settings).
  • Catalogue support (expand on hover).
  • sidedish is just better overall.

It is still suggested you read the install instructions found in the README on the GitHub repo, as it outlines AdBlock/uBlock Origin filters and what is needed/required for ss16 to function correctly.

Overview

Created by saxamaphone !3.saxN0DHY, who has been making userstyles for 4chan since 2009 with styles such as 3 Shades of 4chan (which only exists as Midnight Caek in App/OneeChan now), curabitr, and xl.

ss16 is a fresh attempt to make a userstyle that looks radically different from the typical App/OneeChan aesthetic of today, utilising modern web standards and bleeding edge CSS technologies.

First conceived in December 2015, ss16 has remained more-or-less the same. Updates are few and far between, only really being updated when new 4chan or 4chan X features are implemented.

Features

Documented Code

The unminified version of ss16 contains commented CSS, explaining what it does, and why it does it.

CSS Custom Properties

Variables come to CSS in the form of Custom Properties; ss16 gives you the option to choose not only a primary and secondary colour, but every other colour - only needing to change it once.

Sidedish of Rice

Unlike normal userstyles, ss16 requires a companion userscript for additional rice and features, not found in 4chan X or App/OneeChan.

Fully Featured

While it does enforce some defaults, the latest version of 4chan X will always be supported (and required).

Filtered Fun

Through the use of 4chan X’s filtering feature, additional classes can be added to elements to further enhance ss16.

Web Fonts

ss16 uses @import to load fonts such as Arimo, no longer needing you to download and install fonts locally, or use a Flash loader.

Sensible Defaults

ss16 utilises code from well-established web projects such as Bootstrap, H5BP, Normalize, and the Material Design Guidelines to ensure elements render consistently and correctly across browsers.

No !important;s

As it is meant to be a replacement CSS, ss16 requires users to block both 4chan’s default CSS and 4chan X’s (done with the userscript), resulting in cleaner code.

Material Icons

All icons have been replaced from Font Awesome to Material Icons, and things that should be icons have been turned into them.

It should be noted that ss16 does not function with either the default 4chan Extension or App/OneeChan. 4chan X must only be used.
